    FBV → APIView → GenericAPIView → ViewSet

    (simple)                      (powerful)


// APIVIEW

    "foundation of all class based views the the APIView"

    When a request hits an APIView, DRF automatically does:
        1.Parse request body (JSON, form, multipart)
        2.Authenticate the user
        3.Check permissions
        4.Check throttling
        5.Call your method (get, post, etc.)
        6.Return a proper API response


// The flow 

    "path("api/product/",Product.as_view())"

    as_view() runs a dispatch function that checks prerequities before executing the view.  

    dispatch()
     ├─ initialize_request()
     ├─ initial()
     ├─ call handler (get/post/etc.)
     ├─ handle_exception()
     └─ finalize_response()
